Short Paper: Calory Battle AR: an Extensible

Mobile Augmented Reality Exergame Platform

บทคัดย่อ

          เรานำเสนอเทคโนยีบนโทรศัพท์มือถือที่ผสมผสานระหว่าง  AR  Exergame  Platform   ซึ่งจะใช้  AR  และ เซ็นเซอร์ที่จะเชื่อมต่อกับเกมโลกเสมือนจริงกับโลกแห่งความจริง เราวิเคราะห์สถานะปัจจุบันของ  AR  และการวิจัย  Exergame ได้อธิบายตามแนวคิด Calory  Battle  AR  ประสมกับรายละเอียดย่อยๆทางเทคนิค ในการขยายตัวของแพตฟอร์ม 3 อย่างคือ

1.ผู้เล่นสามารถสร้าง และ แบ่งปันแผ่นที่เกมได้

2.ผู้พัฒนาสามารถสร้างประเภทงานใหม่ๆสำหรับแผนที่เกมได้

3.ผู้พัฒนาสามารถสร้างมอดูล context-sensing ขึ้นอยู่กับเทคโนโลยีที่เหมาะสม

1.บทนำ

          ตามที่องค์การอนามัยโลก ทั่วโลกได้กล่าวว่าโรคอ้วนได้เพิ่มขั้นเป็น2เท่าตั้งแต่ปี  1980  และมากกว่า  40  ล้านคน   เป็น เด็กที่มีอายุต่ำกว่าห้าปีเป็นคนที่มีน้ำหนักเกินในปี  2010  องค์การเพื่อความร่วมมือทางเศรษฐกิจและการพัฒนาหรือ  OECD  ได้ประมาณการในอนาคต สำหรับน้ำหนักเกินและแนะนำแนวทางดำเนินต่อไป   และยังมุ่งเน้นเกี่ยวกับโรคอ้วนในวัยเด็กเพราะกิจกรรมทางกายภาพในวัยเด็กซึ่งเกี่ยวข้องกับกิจกรรมทางกายภาพในวัยผู้ใหญ่ ซึ่งวิธีการหนึ่งที่โดดเด่นในการสร้างแรงจูงใจโดยให้เด็กเล่นเกม  Exergame นี้ ซึ่งเกมนี้รวมการออกกำลังกายกับเกมดิจิตอลและได้แสดงถึงผลผลิตทางกายภาพ ด้านจิตวิทยา ด้านความรู้ และผลประโยชน์ด้านวิชาการ   จนกระทั่งตอนนี้  Exergame จำนวนมากถูกสร้างสำหรับงานเฉพาะ ดังนั้นเราไม่เผยแพร่งานและเป้าหมายที่ต่างกัน เพื่อให้น้อยลง เราจะนำเสนอ  Exergamepaltform ซึ่งใช้เทคโนโลยีที่ผสมผสานระหว่างโลกเสมือนจริงเข้ากับโลกแห่งความเป็นจริง ให้มองเห็นภาพและปฎิสัมพันธ์ของเทคโนโลยี

2.ความเป็นมา

          AR   ได้รับการกำหนดเป็นกรณีสิ่งแวดล้อมจริงรวมกับสิ่งแวดล้อมเสมือน  AR  ความหมายดั้งเดิมคือการเพิ่มให้มองเห็นภาพธรรมชาติที่แท้จริง ไปยังกล้องเพื่อแสดงภาพ ซึ่ง Haller Billinghurstและ Thomas แนะนำ แอพพลิเคชั่นที่ใช้ความต้องการที่จะทราบว่าที่เกิดเหตุตั้งอยู่ในส่วนที่เกี่ยวกับกล้อง   หมายถึง กล้องได้จับวัตถุบางสิ่งซึ่งสามารถเป็นวัตถุจริงๆหรือ  fiducial marker  ซึ่ง  fiducial marker  ทำให้ติดตามงานได้ง่าย และแม่นยำซึ่งแอพพลิเคชั่นรู้ถึงสิ่งที่ต้องการจะมองหา  ตัวอย่างที่ดีของอุปกรณ์ที่ทันสมัยที่ใช้  AR  คือ  Google’s Project Glass

          Moblie  Exergame  จำนวนมากได้เคยถูกนำเสนอเพื่อจูงใจให้ผู้เล่นเคลื่อนที่   ในเกม Zombies Run  คือผู้เล่นต้องวิ่งไปเก็บไอเทมจากพื้นที่นั้นจริงๆ ซึ่งจะถูกบุกรุกโดยซอมบี้  ตัวอย่างอื่น เช่น  Health Defender  ผู้เล่นเกมได้รับแรงบันดาลใจจาก  Space Invaders  ขณะที่กำลังออกกำลังกาย  เกมนี้จะเชื่อมต่อกับตัววัดอัตราการเต้นของหัวใจผู้เล่นจะได้รับโบนัสเมื่ออัตราการเต้นหัวใจของเขาเพิ่มขึ้นอย่างเพียงพอและการเต้นของหัวใจที่สูงพอ  Shakra  คือ กิจกรรม tracker ซึ่งใช้สัญญาณการเปลี่ยนแปลงของ GSM และหน่วยข้อมูลที่เก็บเพื่อใช้ในการติดตามของตำแหน่งผู้ใช้ หลังจากที่ได้รับการฝึกอบรม ผู้ใช้สามารถบอกได้ถูกต้องว่าผู้ใช้จะยังคงยืน เดินหรือวิ่ง ถ้าผู้ใช้กำลังยืน เดิน หรือวิ่ง สามารถบอกข้อมูลได้อย่างแม่นยำ   ใน  GeoBoids  ผู้เล่นจะสำรวจสิ่งต่างๆบนโลกเพื่อค้นหา Geoboids ซึ่งเป็นสิ่งมีชีวิตจำลอง โดยมีเป้าหมายในการจับพวกมันและให้ผู้เล่นได้ออกกำลังกาย   เกมทั้งหมดเหล่านี้  โดยเฉพาะ Geoboids จะนำ  AR  เข้ามาใช้

          จากการวิเคราะห์เบื้องหลัง เกมข้างต้นไม่มีแนวโน้มที่จะแพร่หลาย นอกจากนี้ในผู้เล่นGeoboidsอาจจะไม่ได้รับแรงจูงใจที่จะวิ่งระหว่างผู้เล่นกับฝูงมันโดยการเดิน Health Defender ร้องขอให้ผู้ใช้สวมเครื่องวัดการเต้นของหัวใจ โดยเฉพาะซึ่งสามารถเชื่อมต่อกับโทรศัพท์มือถือโดยบลูทูธShaka เป็นเพียงการติดตามการเคลื่อนไหวภายนอกองค์ประกอบเกม

3.แนวคิด

          Calory  Battle  AR   คือ  mobile  augmented  reality  exergame   ที่สร้างขึ้นบน Android platform  ที่มีจุดมุ่งหมายเพื่อส่งเสริมการออกกำลังกาย หลายexergameไม่ได้สามารถพกพาและจะต้องใช้เล่นภายในบ้านเท่านั้น  (เช่นเกม Nintendo Wii)  ซึ่งแตกต่างกับเกมที่ไม่สามารถเคลื่อนย้ายไปในที่ต่างๆได้   เกม Calory Battle AR จะเชื่อมโยงกับสภาพแวดล้อมจริงและถูกออกแบบมาเพื่อจะเล่นภายนอกไม่มีฮาร์ดแวร์พิเศษที่จำเป็น โดยสิ่งที่จำเป็นของผู้ใช้คือสมาร์ทโฟนและภาพเป้าหมาย

          แผนที่จะเป็นเก็บงานทั้งหมดที่เกี่ยวข้องกับแผนที่เกม ซึ่งรวมถึงรูปแบบและองค์ประกอบ เพื่อใช้พร้อมกับข้อมูลงาน โดยงานหมายถึงกิจกรรมที่ผู้ใช้ต้องการที่จะดำเนินการงานแต่ละงานมีการเชื่อมต่อไปยังเป้าหมายภาพที่ไม่ซ้ำกัน ซึ่งจะนำไปใช้ในโลกแห่งความจริงทำให้งานที่ได้รับมอบหมาย ยังเป็นคู่ประสานงานทางภูมิศาสตร์ ผู้ใช้สามารถสร้างและแชร์แผนที่โดยใช้การแก้ไขเฉพาะจุด

          เมื่อเกมเริ่ม สิ่งแรกที่ผู้ใช้ต้องทำคือการเข้าสู่ระบบหรือสร้างบัญชีโดยนำเสนอเมนูหลัก หากเป็นครั้งแรกในการทำงานของเกม ผู้ใช้จะต้องติดตั้งแผนที่เป็นครั้งแรก ผู้ใช้จะต้องหาและเลือกแผนที่ ไฟล์ข้อมูลแผนที่ที่ได้รับการร้องขอจากเซิร์ฟเวอร์ เมื่อมีการติดตั้งเสร็จแล้วผู้ใช้สามารถเริ่มเกมโดยคลิกที่ปุ่ม new game ในเมนูหลักต่อไปผู้ใช้เลือกแผนที่ที่จะเล่น หลังจากโหลดทรัพยากรเกมจะเริ่มขึ้น ผู้ใช้ครั้งแรกจะได้เห็นมุมมองแผนที่ ซึ่งได้รับการดำเนินการ โดยใช้ Google Maps API v2 สำหรับ Android แผนที่แสดงตำแหน่งปัจจุบันของผู้ใช้และสถานที่งาน โดยการกดปุ่ม information ที่ผู้ใช้จะได้รับข้อมูลเกี่ยวกับแผนที่ปัจจุบัน

          นอกจากนี้โดยการคลิกที่หนึ่งในเป้าหมายบนแผนที่ที่ผู้ใช้จะได้รับข้อมูลเพิ่มเติมเกี่ยวกับงาน เมื่อผู้ใช้พบว่าหนึ่งในงานที่อยู่ในสภาพแวดล้อมจริง เขาจะต้องกดปุ่ม "Start Analyzer!" เพื่อเปลี่ยนมุมมองการวิเคราะห์ ตรวจพบภาพเป้าหมาย และนำเสนอ งานให้ผู้ใช้ที่ได้รับมอบหมายงานนั้น พื้นที่มีสองประเภทงานที่แตกต่างกัน แต่สามารถเพิ่มได้เพราะเป็นสถาปัตยกรรมเป็นยืดหยุ่นได้

4.การออกแบบทางเทคนิคและการดำเนินการ

          Game client  จะดำเนินการเกี่ยวกับ  Android  และจะใช้  Qualcomm’s Vuforia SDK  สำหรับสร้าง  augmented reality  รูปที่ 1   เป็นสถาปัตยกรรมไคลเอนต์ เส้นประที่แสดงโมดูลที่ยังไม่สมบูรณ์   C ++ Code  เป็นการจัดการในส่วนของเนื้อหาที่เป็น  AR  โดยจะมีการตรวจสอบการชน,  การตรวจจับเป้าหมาย,  โหลดแบบจำลอง3D,  เตรียม Vuforia SDK  และการสร้างเหตุการณ์ขึ้นมา   ส่วน Java Code  จะจัดการในส่วนต่างๆ  เช่น  ตรรกะเกมและอินเตอร์เฟซ   การติดต่อกันกับ  C ++ Code  จะใช้  Java Native Interface  และใช้อุปกรณ์ฐานข้อมูลในการจัดเก็บวัตถุแผนที่    HTTP จะใช้สำหรับการเรียกใช้แผนที่จากเซิร์ฟเวอร์  การตรวจสอบกิจกรรม  middleware  จะใช้อุปกรณ์เซ็นเซอร์และข้อมูลตามบริบทอื่น ๆ ที่จะสรุปกิจกรรมของผู้ใช้ (เช่นการทำงาน, การเดิน, การกระโดด) ในเวลาจริง

          Platform  เป็นแบบยืดหยุ่นกับเนื้อหาที่หลากหลายโดยงานจะถูกในเสนอในรูป abstract class  โดยงานแต่ละงานจะสืบทอดมาจาก  task type  งานเหล่านี้สามารถนำไปใช้งานเป็นตัวดักจับเหตุการณ์ เมื่อถูกเรียกขึ้นมาใช้งานในเหตุการณ์ต่างๆเช่น วัตถุ  3  มิติชนกับเหตุการณ์อย่างอื่นแล้วจะให้ทำอะไร เช่นการโชว์ข้อความเหตุการณ์หลัก 6 เหตุการณ์ที่มันดักจับได้  เช่น  เมื่อเริ่มชน  สิ้นสุดการชน  รูปภาพที่วิ่งเข้ามา  รูปภาพหลุดออกจากเฟรม  ในลักษณะการชนจะเด้งเข้า-ออก  เหตุการณ์ที่เกิดร่วมกันสามารถนำข้างต้นดังกล่าวมารวมกัน

5.สรุป

          แพลตฟอร์ม ที่ถูกสร้างขึ้นมาเพื่อรองรับการ  exergaming  แต่ก็มีศักยภาพในด้านอื่น ๆ เช่นกัน การขยายแพลตฟอร์มนี้มี 3 ประการ  ประการแรกผู้ใช้สามารถสร้างและแบ่งปันแผนที่เกมที่ฝังอยู่กับเนื้อหาและงานที่เกี่ยวข้องกับสถานที่เฉพาะและวัตถุประสงค์ นอกจากนี้ยังทำให้ออกแบบให้พกพาแพลตฟอร์มได้  ประการที่สองเกี่ยวข้องกับประการแรกของนักพัฒนาสามารถสร้างประเภทงานใหม่ที่สามารถถูกนำมาใช้ในแผนที่เกมได้  ประการที่สามนักพัฒนาสามารถสร้างและติดตั้งสภาพแวดล้อมใหม่เพื่อสนับสนุนการใช้ ยกตัวอย่างเช่น ช่วงโหว่ของเทคโนโลยีแพลตฟอร์มยังคงมีข้อจำกัดบางอย่างที่จะได้รับการแก้ไข การทำงานร่วมกันระหว่างอุปกรณ์ที่ไม่สามารถทำได้ เกมเบื้องต้นยังขาดคุณสมบัติจำนวนมากและจะสนับสนุนเฉพาะ Android
